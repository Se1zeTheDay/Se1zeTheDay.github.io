最多可以参加的会议数目

题目描述

给你一个数组 events，其中 events[i] = [startDayi, endDayi] ，表示会议 i 开始于 startDayi ，结束于 endDayi 。

你可以在满足 startDayi <= d <= endDayi 中的任意一天 d 参加会议 i 。注意，一天只能参加一个会议。

请你返回你可以参加的 最大 会议数目。

示例 1：

输入：events = [[1,2],[2,3],[3,4]]
输出：3
解释：你可以参加所有的三个会议。
安排会议的一种方案如上图。
第 1 天参加第一个会议。
第 2 天参加第二个会议。
第 3 天参加第三个会议。
示例 2：

输入：events= [[1,2],[2,3],[3,4],[1,2]]
输出：4
示例 3：

输入：events = [[1,4],[4,4],[2,2],[3,4],[1,1]]
输出：4
示例 4：

输入：events = [[1,100000]]
输出：1
示例 5：

输入：events = [[1,1],[1,2],[1,3],[1,4],[1,5],[1,6],[1,7]]
输出：7

提示：

1 <= events.length <= 10^5
events[i].length == 2
1 <= events[i][0] <= events[i][1] <= 10^5

优先队列法

后来看了题解之后有个优先队列的做法，先将会议按照开始时间排好序，然后将开始时间相同的会议添加进优先队列中，为小根堆，那么堆顶即为最早结束的会议，判断会议的结束时间是否在当前天数之前，如果是，那么说明此会议已经过期，否则说明还可以参加，此方法最坏时间复杂度为O(nlogn)。

不过是真的难想到，仔细分析可以发现这个方法遍历的是天数，而不是会议，是根据天数来判断会议是否能参加，因此就不用重复计算，而遍历会议的话，因为会议的时间是有重合的，因此其实我们重复计算了许多。
