递增三元子序列

题目描述

给定一个未排序的数组，判断这个数组中是否存在长度为 3 的递增子序列。

数学表达式如下:

如果存在这样的 i, j, k, 且满足 0 ≤ i < j < k ≤ n-1，
使得 arr[i] < arr[j] < arr[k] ，返回 true ; 否则返回 false 。
说明: 要求算法的时间复杂度为 O(n)，空间复杂度为 O(1) 。

示例 1:

输入: [1,2,3,4,5]
输出: true
示例 2:

输入: [5,4,3,2,1]
输出: false

用变量记录3元序列的前两个

这题的思路十分巧妙，维护了两个变量small与mid，分别记录长为3的递增子序列的最小值和中间值。

在遍历数组时，如果遇到比small小的数，那我们就更新small，遇到比small大比mid小的数时，就更新mid，最后如果遇到比mid大的数，说明已经找到长度为3的递增数组。

在上面的求解过程中，可能会有这样的疑问，比如我们已经找到了长度为2的递增序列，但这个时候又来了一个比small还小的数字，为什么我们可以直接替换small呢，small与mid在原数组中并不是按照索引递增的关系呀？

那么只要想清楚这一点，就明白这个方法了，我们假设当前的small为3，mid为5，这时候来了个1。假如我们不将small替换为1，那么，当下一个数字为2，后面再接上3时，我们无法发现这个[1,2,3]的递增数组，也就是说我们替换最小值是为了后续能够更好地更新中间值，或者换句话说，让small和mid尽可能小，那么第三个数的取值空间就大。

另外，即使我们更新了 small ，这个 small 在 mid 后面，没有严格遵守递增顺序，但它隐含着的真相是，有一个比 small 大比 mid 小的前·最小值出现在 mid 之前。因此，当后续出现比 mid 大的值的时候，我们一样可以通过当前 small 和 mid 推断的确存在着长度为 3 的递增序列。 所以，这样的替换并不会干扰我们后续的计算！

前后遍历

定义两个数组forward[i]和backward[i]，forward[i]从前向后遍历，保存从0-i的最小元素，backward[i]从后往前遍历，记录[i - size - 1]之间的最大元素值。然后从前向后遍历，如果找到一个数满足forward[i] < nums[i] < backward[i]，则说明三元子序列存在。
