零钱兑换

题目描述

给定不同面额的硬币 coins 和一个总金额 amount。编写一个函数来计算可以凑成总金额所需的最少的硬币个数。如果没有任何一种硬币组合能组成总金额，返回 -1。

示例 1:

输入: coins = [1, 2, 5], amount = 11
输出: 3
解释: 11 = 5 + 5 + 1
示例 2:

输入: coins = [2], amount = 3
输出: -1

说明:
你可以认为每种硬币的数量是无限的。

题目分析

这类题还是不怎么会做，每次遇到不会做的还不想学，究竟是没有兴趣呢还是懒呢？

自下而上的动态规划解法

动态规划问题具有最优子结构特性，我们通过这个特性来建立状态转移方程。
针对这道题，我们定义F(i)为组成金额i所需的最少的硬币数量，那么在计算F(i)时，我们已经知道了F(0) ~ F(i - 1)的值。通过思考，我们可以发现F(i)只能来源于F(i - c(j)),c(j)代表第j种硬币的面额，而我们需要做的是最小化硬币数量，因此列出的转移方程如下：
F(i) = min F(i - c(j)) + 1, 0 <= j <= n - 1

有了转移方程之后，代码就好写了，注意这里有min操作，因此我们初始化数组时将值放的大一些，以免影响判断。
