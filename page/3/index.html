只出现一次的数字

题目描述

给定一个非空整数数组，除了某个元素只出现一次以外，其余每个元素均出现两次。找出那个只出现了一次的元素。

说明：

你的算法应该具有线性时间复杂度。 你可以不使用额外空间来实现吗？

示例 1:

输入: [2,2,1]
输出: 1
示例 2:

输入: [4,1,2,1,2]
输出: 4

题目分析

这个题目要求时间复杂度为O(n),说实话我一点思路都没有，解法真的难想。

如果没有空间复杂度O(1)的限制，那还有挺多方法。

这里简单列举几个：

使用集合

1，使用集合(set)存储数字,遍历数组中的每个数字，如果集合中没有该数字，则将该数字加入集合，否则在集合中删除该元素，那么最后集合中剩下的一个元素就是只出现一次的元素。
2，集合取差值得到只出现一次的元素，遍历数组中的每个数字，如果集合中没有该数字，则将该数字加入集合，否则加入另外一个集合，最后将两个集合做差值得到改元素。

使用哈希表

使用哈希表存储每个数字和该数字出现的次数，遍历哈希表即得到只出现一次的数字。

此题符合题意的解:位运算

首先说明位运算的三个性质：
1，任何数和0做异或运算，结果仍然为原来的数，即a ⊕ 0 = a。
2，任何数和其自身做异或运算，结果仍然是0，即a ⊕ a = 0。
3，异或运算满足交换律和结合律，即 a ⊕ b ⊕ a = b ⊕ a ⊕ a = b ⊕ (a ⊕ a) = b ⊕ 0 = b。

那么接下来就很容易了，只需要将数组中的所有元素进行异或运算，剩下来的那个元素就是只出现一次的元素。

命令英文原意：GNU zip
命令所在路径：/bin/gzip

功能描述：压缩文件(不能压缩目录)

命令英文原意：GNU unzip
命令所在路径：/bin/gunzip
执行权限:所有用户
语法：gunzip [压缩文件]
功能描述：解压缩.gz的压缩文件
拓展：使用 gzip -d同理

tar

命令所在路径：/bin/tar
执行权限：所有用户
语法：tar 选项[-z -c -f -v] [压缩后文件名] [目录]
-c 打包
-v 显示详细信息
-f 指定文件名
-z 打包同时压缩
-j 打包同时压缩bzip2
功能描述：打包目录
压缩后文件格式：.tar.gz
拓展：打包和压缩的区别
相同点：打包和压缩都可以将多个文件生成一个文件存储
不同点：压缩操作不但要生成一个文件，还要通过压缩工具对文件进行压缩，来减少文件占用的存储空间大小。

tar命令解压缩

-x 解包
-v 显示详细信息
-f 指定压缩文件
-z 解压缩
-j 解压缩bzip2
例：tar -zxvf Japan.tar.gz

zip

命令所在路径：/usr/bin/zip
执行权限：所有用户
语法：zip [-r] [压缩后的文件名] [需要压缩的文件或目录]
-r 压缩目录
功能描述： 压缩文件或目录
压缩后文件格式：.zip
解压缩：unzip

bzip2

命令所在路径：/usr/bin/bzip2
执行权限：所有用户
语法：bzip [-k] [文件]
-k 产生压缩文件后保留原文件
功能：压缩文件(推荐压缩较大文件)
压缩后文件格式：.bz2
范例：tar -cjf ***.tar.bz2 ***
解压缩： bunzip2 -k [文件名.bz2]
tar -xjf [文件名.tar.bz2]

关机重启命令

shutdown

语法：shutdown [选项] 时间选项
选项： -c 取消前一个关机命令
-h 关机(halt)
-r 重启(reboot)
例：shutdown -h now
shutdown -r now
拓展：
其他关机命令：halt poweroff init 0
其他重启命令：reboot init 6
系统运行级别：0 关机
1 单用户
2 不完全多用户，不含NFS服务(网络文件系统，用于两个linux之间的文件共享)
3 完全多用户
4 未分配
5 图形界面
6 重启
退出登录命令：logout

rpm -ivh 包全名

rpm -Uvh 包全名

rpm包卸载

rpm -e 包名
选项： -e(erase) 卸载

选项： -a 所有(all)
