堆树(最大堆，最小堆)

一、堆树的定义

(1) 堆树是一棵完全二叉树
(2) 堆树中某个节点的值总是不大于(最小堆)或不小于(最大堆)其孩子节点的值
(3) 堆树中的每个节点的子树都是堆树

当父节点的键值总是大于或等于任何一个子节点的键值时为最大堆，也称大根堆。
当父节点的键值总是小于或等于任何一个子节点的键值时为最小堆，也称小根堆。
如图所示，上边堆树为最大堆，下边为最小堆。

2.2 构造最大堆

在构造堆时的基本思想就是：首先将每个叶子节点视为一个堆，再将每个叶子节点与其父节点一起构造成一个包含更多节点的堆。

因此，在构造堆时，我们首先需要找到最后一个节点的父节点，也就是最后一个具有孩子的节点，从这个节点开始构造最大堆；直到该节点前面所有分支节点都处理完毕，那么最大堆就构造完毕了。

假设树的节点个数为n，以1为下标开始编号，直到n结束。对于节点i，其父节点为i/2，左孩子节点为i * 2，右孩子节点为2 * i + 1。

颜色分类

题目描述

给定一个包含红色、白色和蓝色，一共 n 个元素的数组，原地对它们进行排序，使得相同颜色的元素相邻，并按照红色、白色、蓝色顺序排列。

此题中，我们使用整数 0、 1 和 2 分别表示红色、白色和蓝色。

注意:
不能使用代码库中的排序函数来解决这道题。

示例:

输入: [2,0,2,1,1,0]
输出: [0,0,1,1,2,2]

进阶：

一个直观的解决方案是使用计数排序的两趟扫描算法。
首先，迭代计算出0、1 和 2 元素的个数，然后按照0、1、2的排序，重写当前数组。
你能想出一个仅使用常数空间的一趟扫描算法吗？

题目分析

这个题目我记得就是荷兰国旗问题，可以用时间复杂度O(n),空间复杂度O(1)解决，但是我忘记具体是怎么做的了，唉。。。

于是乎又开始看题解了，一看这个问题竟然是Dijkstra提出的，不得不佩服他，计算机领域的许多方面都有他的身影。

接下来说一下具体的思路：
如图所示，less和more指针其实是用来划分三个区域的，小于区，等于区和大于区。

我们初始化less = -1，more = arr.length，当前位置为cur = 0。
那么分为如下四种情况：
· 如果arr[cur] < num，那么交换arr[cur]和arr[++less]，接着cur++

· 如果arr[cur] > num，那么交换arr[cur]和arr[–more]，这时cur不变

· 如果当前位置上的数为num，也就是为等于区，那么less与more均不变，只需将cur++

· 最后当cur == more时，停止循环

这里你可能会有疑问，为什么第一种情况cur需要++，而第二种不需要，原因是第一种交换过来的元素之前已经被cur遍历过，一定小于等于num,因此不需要再次判断，于是cur++，而后面的元素则没有遍历过，交换过来的元素需要再次判断。

105-从前序与中序遍历序列构造二叉树

2020-07-18
leetcode

从前序与中序遍历序列构造二叉树

题目描述

根据一棵树的前序遍历与中序遍历构造二叉树。

注意:
你可以假设树中没有重复的元素。

例如，给出

前序遍历 preorder = [3,9,20,15,7]
中序遍历 inorder = [9,3,15,20,7]
返回如下的二叉树：
3
/
9 20
/
15 7

题目分析

这道题本能反应就是用递归，因为左右子树的处理与原树的处理相同。

在做这道题之前我们复习一下二叉树遍历的顺序：

二叉树前序遍历的顺序为：

·先遍历根节点；

·随后递归地遍历左子树；

·最后递归地遍历右子树。

二叉树中序遍历的顺序为：

·先递归地遍历左子树；

·随后遍历根节点；

·最后递归地遍历右子树。

在「递归」地遍历某个子树的过程中，我们也是将这颗子树看成一颗全新的树，按照上述的顺序进行遍历。挖掘「前序遍历」和「中序遍历」的性质，我们就可以得出本题的做法。

递归

对于任意一颗树而言，前序遍历的形式总是

[ 根节点, [左子树的前序遍历结果], [右子树的前序遍历结果] ]

即根节点总是前序遍历中的第一个节点。而中序遍历的形式总是

[ [左子树的中序遍历结果], 根节点, [右子树的中序遍历结果] ]

只要我们在中序遍历中定位到根节点，那么我们就可以分别知道左子树和右子树中的节点数目。由于同一颗子树的前序遍历和中序遍历的长度显然是相同的，因此我们就可以对应到前序遍历的结果中，对上述形式中的所有左右括号进行定位。

这样以来，我们就知道了左子树的前序遍历和中序遍历结果，以及右子树的前序遍历和中序遍历结果，我们就可以递归地对构造出左子树和右子树，再将这两颗子树接到根节点的左右位置。

细节

注意这里有个细节可以注意到，我们在中序遍历数组中定位根节点时，可以提前将中序遍历数组元素及其下标记录在哈希表中，这样一来就可以减少查找的空间复杂度至O(1)。
