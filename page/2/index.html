题目分析

这类题还是不怎么会做，每次遇到不会做的还不想学，究竟是没有兴趣呢还是懒呢？

自下而上的动态规划解法

动态规划问题具有最优子结构特性，我们通过这个特性来建立状态转移方程。
针对这道题，我们定义F(i)为组成金额i所需的最少的硬币数量，那么在计算F(i)时，我们已经知道了F(0) ~ F(i - 1)的值。通过思考，我们可以发现F(i)只能来源于F(i - c(j)),c(j)代表第j种硬币的面额，而我们需要做的是最小化硬币数量，因此列出的转移方程如下：
F(i) = min F(i - c(j)) + 1, 0 <= j <= n - 1

有了转移方程之后，代码就好写了，注意这里有min操作，因此我们初始化数组时将值放的大一些，以免影响判断。

X X X X

这个方法妙就妙在它想到建立虚拟节点,这里可以总结一下就是用并查集区分两个集合可以通过区分根节点，也可以通过判断是否与虚拟节点进行连接。这个地方明显区分根节点比较麻烦，因为内部的不同区域根节点虽然不同，但是最终都需要改变成X。

leetcode

合并区间

题目描述

给出一个区间的集合，请合并所有重叠的区间。

示例 1:

输入: [[1,3],[2,6],[8,10],[15,18]]
输出: [[1,6],[8,10],[15,18]]
解释: 区间 [1,3] 和 [2,6] 重叠, 将它们合并为 [1,6].
示例 2:

输入: [[1,4],[4,5]]
输出: [[1,5]]
解释: 区间 [1,4] 和 [4,5] 可被视为重叠区间。

题目分析

这道题我一开始没有想排序，想的是记录区间的开始和结束的位置，定义一个flag数组，区间覆盖的范围就标1，那么无论重合不重合，只要是1就可以了，但是这个方法有个问题，就是像[1,3][4,5]这种情况1,2,3,4,5都是1，因此无法判断。

然后可以进行改进，改进的方法是这样的，设一个区间为[m, n]，那么我们就置flag[m] = 1, flag[n] = -1,所以不管有没有重合，只要flag数组中和为0，就是一个完整的视频。
这种方法不像上一种一样有致命性的缺陷，只有一些小缺点，比如有区间为首尾重合的时候，那么这个区间就会丢失。如[[1,3][0,0]]，这时候[0,0]这个区间就丢掉了。

因此我们也不采用这个方法，所以我们介绍排序法。

排序法

具体思路就是数组排好序之后我们只需要比较数组的右端就可以了，将当前数组与已合并好区间的数组中的最后一个数组进行比较，记当前数组为cur[],记最后一个数组为peek[]。

若cur[0] > peek[1],那么两个区间不重合，直接添加进合并好区间的数组中。
若cur[0] < peek[1],那么两个区间有重合的部分，这时我们需要取两个数组右端更大的那个数，即max{cur[1], peek[1]}。

·题目数据保证答案唯一，换句话说，数组中前 k 个高频元素的集合是唯一的。
·你可以按任意顺序返回答案。

这里介绍大体思路：
·我们先借助哈希表来建立数字和其出现次数的映射，即遍历一遍数组统计元素的频率。
·因为最小堆的堆顶永远是当前堆中的最小元素，所以当我们遇到比堆顶元素频率大的元素，就弹出堆顶元素，然后将新元素添加进堆中。
·最终，堆中的k个元素即为前k个高频元素
