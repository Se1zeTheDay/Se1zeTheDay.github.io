380-常数时间插入、删除和获取随机元素

2020-07-31
leetcode

常数时间插入、删除和获取随机元素

题目描绘

设计一个支持在平均 时间复杂度 O(1) 下，执行以下操作的数据结构。

insert(val)：当元素 val 不存在时，向集合中插入该项。
remove(val)：元素 val 存在时，从集合中移除该项。
getRandom：随机返回现有集合中的一项。每个元素应该有相同的概率被返回。
示例 :

// 初始化一个空的集合。
RandomizedSet randomSet = new RandomizedSet();

// 向集合中插入 1 。返回 true 表示 1 被成功地插入。
randomSet.insert(1);

// 返回 false ，表示集合中不存在 2 。
randomSet.remove(2);

// 向集合中插入 2 。返回 true 。集合现在包含 [1,2] 。
randomSet.insert(2);

// getRandom 应随机返回 1 或 2 。
randomSet.getRandom();

// 从集合中移除 1 ，返回 true 。集合现在包含 [2] 。
randomSet.remove(1);

// 2 已在集合中，所以返回 false 。
randomSet.insert(2);

// 由于 2 是集合中唯一的数字，getRandom 总是返回 2 。
randomSet.getRandom();

题目分析

题目需要三个操作：插入、删除、随机取元素都在时间复杂度O(1)，那么如果我们使用哈希表,能够满足前两个操作的要求但是不能满足随机取元素的时间复杂度的要求，如果我们使用动态数组，那么删除元素不能满足，因此我们准备把两者结合起来。

我们使用动态数组保存数据，并使用哈希表保存动态数组中数据与下标的映射，那么在删除操作时我们只需要将需要删除的元素与最后一个元素进行交换，最后修改哈希表中对应的下标值即可。

阶乘后的零

题目描述

给定一个整数 n，返回 n! 结果尾数中零的数量。

示例 1:

输入: 3
输出: 0
解释: 3! = 6, 尾数中没有零。
示例 2:

输入: 5
输出: 1
解释: 5! = 120, 尾数中有 1 个零.
说明: 你算法的时间复杂度应为 O(log n) 。

题目分析

这道题涉及到了一些初等数论的知识，比如整数的唯一分解定理。

一个整数一定可以被分解成唯一质数乘积的组合。

那么我们查看一个数阶乘运算后尾数有多少零只需要查看质因子中有多少2与5的组合。

因此例如n!,分别计算1 ~ n - 1的质因子并统计因子2与因子5的个数，最后确定对数即为0的个数。

接着我们可以发现含有因子2的数每两个就出现一次，含有因子5的数每5个出现一次，因此后者要比前者数量少得多，我们只需要统计后者。

但是这样时间复杂度依然是O(n),不符合要求，我们继续探究可以发现。

既然含有因子5的数每5个出现一次，那么n/5就表示含有因子5的数的个数，但是不要忘记，像25，50这样的数有两个5，那么总结一下就是每隔5个数出现一个5，每隔25个数出现两个5，每隔125个数出现3个5。。。以此类推。

那么最终5的个数就是n/5 + n/25 + n/125 + …，注意为什么不是2*n/25，因为在n/5中已经算过一个5了，因此只要再算一个就可以了。

自下而上的动态规划解法

动态规划问题具有最优子结构特性，我们通过这个特性来建立状态转移方程。
针对这道题，我们定义F(i)为组成金额i所需的最少的硬币数量，那么在计算F(i)时，我们已经知道了F(0) ~ F(i - 1)的值。通过思考，我们可以发现F(i)只能来源于F(i - c(j)),c(j)代表第j种硬币的面额，而我们需要做的是最小化硬币数量，因此列出的转移方程如下：
F(i) = min F(i - c(j)) + 1, 0 <= j <= n - 1

有了转移方程之后，代码就好写了，注意这里有min操作，因此我们初始化数组时将值放的大一些，以免影响判断。

运行你的函数后，矩阵变为：

X X X X
X X X X
X O X X

解释:

被围绕的区间不会存在于边界上，换句话说，任何边界上的 ‘O’ 都不会被填充为 ‘X’。 任何不在边界上，或不与边界上的 ‘O’ 相连的 ‘O’ 最终都会被填充为 ‘X’。如果两个元素在水平或垂直方向相邻，则称它们是“相连”的。

题目分析

今天把并查集又看了一遍，想着找个并查集的题目练练手，就按着tag搜到了这道题。

这道题和岛屿数量很相似，不同的地方就是这题的边界不能算被围绕的，因此我们需要区分内部的O和与边界相连的O。

我的想法是相邻的O会合并，合并之后最终这些O的根节点都相同，之后我们建立一个标志数组，如果这篇区域中有在边缘的O，那么我们置标志数组中下标为这片区域的根节点的元素为true，标志着这片区域之后不用换成X，但是这个方法有个问题，就是合并之后根节点会根据union函数的规则进行改变，因此原来的true到最终可能会丢失，这个方法能ac，但是代码很繁琐，这里我就将它贴在文章最后了。

接下来我们介绍另一种很妙的方法。

并查集建立虚拟节点

这个方法妙就妙在它想到建立虚拟节点,这里可以总结一下就是用并查集区分两个集合可以通过区分根节点，也可以通过判断是否与虚拟节点进行连接。这个地方明显区分根节点比较麻烦，因为内部的不同区域根节点虽然不同，但是最终都需要改变成X。

个人觉得并查集不能作为搜索算法，只能与循环一同才可以，而且循环的顺序是固定的，因此我们不能像下面dfs和bfs那样只需要搜索边界O。

我们的思路是把所有边界上的 O 看做一个连通区域。遇到 O 就执行并查集合并操作，这样所有的 O 就会被分成两类

·和边界上的 O 在一个连通区域内的。这些 O 我们保留。
·不和边界上的 O 在一个连通区域内的。这些 O 就是被包围的，替换。
